Detailed Description

Timers are single shot.

Timer times are in milliseconds.

Note
that timer callback functions are called from an arbitrary thread context. The expiration function should do its work as quickly as possible; blocking should be avoided.
On Windows, the separate function vcos_timer_init() must be called as timer initialization from DllMain is not possible.

◆ vcos_timer_init()

VCOSPRE_ VCOS_STATUS_T VCOSPOST_ vcos_timer_init ( void  )

Perform timer subsystem initialization. This function is not needed on non-Windows platforms but is still present so that it can be called. On Windows it is needed because vcos_init() gets called from DLL initialization where it is not possible to create a time queue (deadlock occurs if you try).

Returns
VCOS_SUCCESS on success. VCOS_EEXIST if this has already been called once. VCOS_ENOMEM if resource allocation failed.
