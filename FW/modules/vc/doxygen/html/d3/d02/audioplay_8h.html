Detailed Description

This API allows the host to provide PCM samples to be rendered via audio_render.

This file describes a simple API for host applications to play sound using VideoCore. It includes the functionality to:

  • open/close
  • set pcm parameters
  • set buffer size parameters
  • retrieve empty buffer available to use
  • send full buffer to be played
  • retrieve current buffering level

This API has no thread context of it's own, so the caller must be aware that the IL API will be used in context. This has implications on executing calls inside callback contexts, and on the minimum size of stack that the caller requires. See the ilclient_stack_size() function for assistance.

This API will use a single audio_render IL component, and supply buffers to the input port using the OpenMAX IL base profile mode.

The audioplay_create() function creates the audioplay object.

Parameters
handleOn success, this is filled in with a handle to use in other API functions.
sample_rateThe sample rate, in samples per second, for the PCM data. This shall be between 8000 and 96000.
num_channelsThe number of channels for the PCM data. Must be 1, 2, 4, or 8. Channels must be sent interleaved.
bit_depthThe bitdepth per channel per sample. Must be 16 or 32.
num_buffersThe number of buffers that will be created to write PCM samples into.
buffer_sizeThe size in bytes of each buffer that will be used to write PCM samples into. Note that small buffers of less than a few Kb in size may be faster than larger buffers, although this is platform dependent.
Returns
0 on success, -1 on failure.
