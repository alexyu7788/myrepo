◆ pf_read

VC_CONTAINER_STATUS_T(* VC_CONTAINER_PRIVATE_T::pf_read) (VC_CONTAINER_T *context, VC_CONTAINER_PACKET_T *packet, VC_CONTAINER_READ_FLAGS_T flags)

Reads a data packet from a container reader. By default, the reader will read whatever packet comes next in the container and update the given VC_CONTAINER_PACKET_T structure with this packet's information. This behaviour can be changed using the VC_CONTAINER_READ_FLAGS_T.
VC_CONTAINER_READ_FLAG_INFO will instruct the reader to only return information on the following packet but not its actual data. The data can be retreived later by issuing another read request. VC_CONTAINER_READ_FLAG_FORCE_TRACK will force the reader to read the next packet for the selected track (as present in the VC_CONTAINER_PACKET_T structure) instead of defaulting to reading the packet which comes next in the container. VC_CONTAINER_READ_FLAG_SKIP will instruct the reader to skip the next packet. In this case it isn't necessary for the caller to pass a pointer to a VC_CONTAINER_PACKET_T structure unless the VC_CONTAINER_READ_FLAG_INFO is also given. A combination of all these flags can be used.

Parameters
contextPointer to the context of the reader to use
packetPointer to the VC_CONTAINER_PACKET_T structure describing the data packet This needs to be partially filled before the call (buffer, buffer_size)
flagsFlags controlling the read operation
Returns
the status of the operation
