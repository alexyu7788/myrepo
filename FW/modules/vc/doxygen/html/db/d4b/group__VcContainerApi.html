Specifies that we're done adding new tracks. This is optional but can be used by writers to trigger the writing of the container header early. If this isn't used, the header will be written when the first data packet is received.
No arguments.
return= VC_CONTAINER_ERROR_TRACK_FORMAT_NOT_SUPPORTED if the format is not supported

Change the format of a track in the list of tracks. This should be used by writers to modify the format of a track at run-time.
Arguments:
arg1= unsigned int: index of track to change
arg2= VC_CONTAINER_ES_FORMAT_T *: format of the track to add
return= VC_CONTAINER_ERROR_TRACK_FORMAT_NOT_SUPPORTED if the format is not supported

VC_CONTAINER_CONTROL_TRACK_DEL 

Deletes a track from the list of tracks. This should be used by writers to delete tracks during run-time. Note that vc_container_close will automatically delete all track so it isn't necessary to call this before closing a writer.
Arguments:
arg1= index of the track to delete

VC_CONTAINER_CONTROL_DRM_PLAY 

Activate the playback of DRM protected content.
No arguments.
return= one of the VC_CONTAINER_ERROR_DRM error codes if content can't be played

Reads a data packet from a container reader. By default, the reader will read whatever packet comes next in the container and update the given VC_CONTAINER_PACKET_T structure with this packet's information. This behaviour can be changed using the VC_CONTAINER_READ_FLAGS_T.
VC_CONTAINER_READ_FLAG_INFO will instruct the reader to only return information on the following packet but not its actual data. The data can be retreived later by issuing another read request.
VC_CONTAINER_READ_FLAG_FORCE_TRACK will force the reader to read the next packet for the selected track (as present in the VC_CONTAINER_PACKET_T structure) instead of defaulting to reading the packet which comes next in the container.
VC_CONTAINER_READ_FLAG_SKIP will instruct the reader to skip the next packet. In this case it isn't necessary for the caller to pass a pointer to a VC_CONTAINER_PACKET_T structure unless the VC_CONTAINER_READ_FLAG_INFO is also given.
A combination of all these flags can be used.
